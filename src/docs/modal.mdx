# Modal

## O que é
Overlay que aparece sobre a tela atual para capturar confirmações, formulários curtos ou mensagens importantes.

## Capacidades
- Declarar `modal Nome:` acessível por botões/links (`@[Abrir modal](Nome)`).
- Conter qualquer conteúdo válido: `card`, `stack`, inputs e botões.
- Fechar explicitamente com ações `@ghost[Cancelar](close)` ou navegando para outra tela.
- Integrar com `list`, `component` ou `fab` para fluxos contextuais.

## Sintaxe
- `modal ConfirmDelete:`
- `card:`
- `# Confirmar exclusão`
- `> Esta ação não pode ser desfeita.`
- `row-end:`
- `@ghost[Cancelar](close)`
- `@destructive[Excluir](delete)`


## Variantes & Slots
| Variante/Slot | Papel | Notas |
| --- | --- | --- |
| `modal Nome:` | Define overlay | Nome inicia com letra maiúscula. |
| Conteúdo interno | Slot | Normalmente envolve `card` para padronizar padding. |

## Casos de uso
- Confirmar ações destrutivas (`modal ConfirmDelete`).
- Capturar dados rápidos (ex.: formulário de convite com 1–2 campos).
- Exibir mensagens e etapas adicionais sem navegar para outra tela.

## Exemplos
<DslPreview title="Exemplo 1">
```dsl
modal ConfirmDelete:
  card:
    # Confirmar exclusão
    > Esta ação não pode ser desfeita.
    row-end:
      @ghost[Cancelar](close)
      @destructive[Excluir agora](deleteItem)

screen DangerZone:
  container:
    # Configurações avançadas
    card:
      *> Ao excluir, todos os dados serão removidos.
      @destructive[Excluir conta](ConfirmDelete)
```
</DslPreview>

<DslPreview title="Exemplo 2">
```dsl
modal InviteMember:
  card:
    # Convidar novo membro
    ___email: E-mail{usuario@empresa.com} | required
    row-end:
      @ghost[Cancelar](close)
      @success[Enviar convite](sendInvite)

screen Team:
  container:
    # Equipe
    list:
      - Ana Souza | Product Manager
      - Bruno Lima | Engineering Manager

    @[Adicionar membro](InviteMember)
```
</DslPreview>
