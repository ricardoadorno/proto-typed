# Mobile

Mobile components provide mobile-friendly primitives like `header:`, `navigator:`, and `fab {icon}`.

## Header

<DslExample
	title="Mobile Header"
	heightClassName="h-56"
	code={`
screen Inbox:
	header:
		# Inbox
	> Messages
	`}
/>

## Navigator

<DslExample
	title="Bottom Navigator"
	heightClassName="h-72"
	description="Use - [label]{icon}(action) items inside navigator or drawer."
	code={`
screen Home:
	header:
		# Home
	navigator:
		- [Home]{home}(Home)
		- [Contacts]{users}(Contacts)
		- [Settings]{settings}(Settings)

screen Contacts:
	header:
		# Contacts
	> People list
	`}
/>

## Floating Action Button (FAB)

<DslExample
	title="FAB"
	heightClassName="h-64"
	code={`
screen Tasks:
	header:
		# Tasks
	list:
		- Task A
		- Task B
	fab {plus}
	`}
/>

Tips:

- Use `header:` for top app bars on mobile screens.
- `navigator:` can be combined with screens to drive navigation.
- `fab {icon}` works best when paired with lists or primary actions.
