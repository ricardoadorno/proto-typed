# Component Composition

## O que é
Composição e boas práticas de reuso combinando componentes menores para formar experiências completas.

## Sintaxe
```proto
component Wrapper:
  $InnerComponent
```

Exemplo
<DslPreview>
```dsl
component HeaderBar:
  header:
    # %title
    row-end:
      @ghost[i-Search Buscar](Search)
      @ghost[i-Settings Config](Settings)

component ProjectCard:
  card:
    ## %name
    >>> %owner
    > %description
    @[Abrir](%target)

component ProjectSection:
  container:
    $HeaderBar:
      - %title
    grid-3 $ProjectCard:
      - Lançamento app mobile | Ana Souza | Status: Em QA | ProjectDetails
      - Campanha Q4 | Bruno Lima | Planejamento finalizando | CampaignPlan
      - Pesquisa usuários | Carla Dias | Entrevistas marcadas | ResearchBoard

screen Dashboard:
  $ProjectSection:
    - Projetos ativos
```
</DslPreview>

## Variações e Props
| Conceito | Tipo | Descrição | Exemplo |
| --- | --- | --- | --- |
| `component dentro de component` | composição | Permite reutilizar blocos inteiros | `$HeaderBar` dentro de `ProjectSection` |
| `prop forwarding` | fluxo | Passe props do componente externo para o interno | `$ProjectSection: - Projetos ativos` |
| `component wrapper` | padrão | Encapsule vários elementos em um único componente pai | `component ProjectSection:` |
| `component + list` | binding | Liste componentes compostos para dashboards complexos | `grid-3 $ProjectCard:` |
| `helper components` | organização | Crie componentes utilitários para cabeçalhos, barras e seções | `component HeaderBar:` |

## Boas práticas
- Separe componentes em níveis (layout, elemento, seção) para reduzir acoplamento.
- Documente no código quais props são obrigatórias para cada composição.
- Prefira composições rasas (até 3 níveis) para facilitar manutenção.

## Erros comuns
⚠ Acessar props inexistentes (`%algo`) gera string vazia no preview.
⚠ Encadear componentes sem limites pode dificultar debug do layout final.
