# Floating Action Button (FAB)

## O que é
Botão flutuante fixado sobre o conteúdo que expõe uma ação global importante — como criar registros ou abrir painéis rápidos.

## Capacidades
- Declarar FAB com um item (`- i-Icon | Destino`) ou várias ações (múltiplas linhas).
- Acionar telas, modais ou drawers, usando nome do destino na segunda coluna.
- Complementar a UI com ícones da biblioteca Lucide (`i-Plus`, `i-Filter`).
- Aparecer em qualquer `screen`, normalmente após blocos principais.

## Sintaxe
- `fab:`
- `- i-Plus | NovaTela`
- `- i-Filter | FilterDrawer`


## Variantes & Slots
| Variante/Slot | Papel | Notas |
| --- | --- | --- |
| Linha `- i-Icon | Destino` | Registrar ação | `Destino` pode ser `Screen`, `Modal` ou `Drawer`. |
| Múltiplas linhas | Menu flutuante | A primeira ação é a principal; demais são secundárias. |
| Ícone (`i-`) | Indicação visual | Utilize nomes válidos da biblioteca Lucide. |

## Casos de uso
- `fab:` com apenas `i-Plus` para criar itens rapidamente.
- `fab:` com `i-Filter` abrindo um `drawer` com filtros avançados.
- `fab:` em mobile para destacar a ação principal do fluxo.

## Exemplos
<DslPreview title="Exemplo 1">
```dsl
screen Tasks:
  container:
    # Tarefas
    list:
      - Revisar backlog | @[Editar](TaskForm)
      - Preparar release | @[Editar](TaskForm)

  fab:
    - i-Plus | TaskForm
```
</DslPreview>

<DslPreview title="Exemplo 2">
```dsl
drawer FilterDrawer:
  container:
    # Filtros avançados
    ___select: Status{Todos | Em andamento | Concluído}
    ___select: Responsável{Ana | Bruno | Carla}
    row-end:
      @ghost[Cancelar](close)
      @primary[Aplicar filtros](applyFilters)

screen Dashboard:
  container:
    # Visão geral
    > Use o FAB para abrir filtros ou criar ações rápidas.

  fab:
    - i-Plus | TaskForm
    - i-Filter | FilterDrawer
```
</DslPreview>
