# Component Definition

## O que é
Declaração de um componente reutilizável (`component Nome:`) que encapsula layout e lógica visual para ser instanciado com `$Nome`.

## Capacidades
- Definir estrutura base usando qualquer layout (card, stack, row, grid).
- Inserir placeholders `%prop` interpolados quando o componente é utilizado.
- Reaproveitar o mesmo componente em múltiplas telas e listas.
- Agrupar conteúdo complexo em um bloco único para manutenção mais simples.

## Sintaxe
- `component Hero:`
- `card:`
- `# %title`
- `> %subtitle`
- `@[Ação](%destination)`
- `screen Home:`
- `container:`
- `$Hero:`
- `- Bem-vindo | Conecte-se rapidamente | Dashboard`


## Variantes & Slots
| Variante/Slot | Papel | Notas |
| --- | --- | --- |
| `component Nome:` | Declaração | Nome em PascalCase. |
| `%prop` | Placeholder | Recebe valores passados na instância. |
| `$Nome` | Instância simples | Usa conteúdo fixo definido no componente. |
| `$Nome:` + lista | Instância com props | Cada linha `-` envia valores separados por `|`. |

## Casos de uso
- Criar cards padrão para produtos, métricas ou contatos.
- Centralizar o layout de uma linha de lista e reaproveitar via `list $Nome:`.
- Isolar seções hero (título, descrição, CTA) repetidas em diversas telas.

## Exemplos
<DslPreview title="Exemplo 1">
```dsl
component HeroBanner:
  card:
    ## %title
    > %subtitle
    @[Saiba mais](%destination)

screen Home:
  container:
    # Bem-vindo(a)
    $HeroBanner:
      - Prototipe mais rápido | Construa telas em minutos | Guides

screen Guides:
  container:
    # Biblioteca de guias
    > Explore fluxos prontos para acelerar a prototipagem.
```
</DslPreview>

<DslPreview title="Exemplo 2">
```dsl
component MetricCard:
  card:
    ## %label
    # %value
    >> %trend

screen Dashboard:
  container-wide:
    grid-3:
      $MetricCard:
        - Leads captados | 1.240 | +8%
      $MetricCard:
        - Conversões | 134 | +5%
      $MetricCard:
        - Satisfação | 92% | +3%
```
</DslPreview>
