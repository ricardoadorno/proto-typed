# Themes
## Por que importa
Temas adaptam o produto à identidade visual da marca, garantindo contraste adequado e experiência consistente em todas as telas. Ajustar tokens core evita intervenção manual em cada componente.

## Forma canônica (sintaxe mínima)
<DslPreview title="Tema customizado no bloco de estilos">
```dsl
styles:
  --primary: oklch(0.68 0.15 210);
  --primary-foreground: oklch(0.98 0 0);
  --background: oklch(0.14 0.01 240);
  --foreground: oklch(0.95 0 0);
```
</DslPreview>
Por quê: define cores principais e de texto, base para qualquer variação temática.

## Padrões de uso
<DslPreview title="Paleta da marca aplicada a planos de assinatura">
```dsl
styles:
  --primary: oklch(0.7 0.15 220);
  --accent: oklch(0.68 0.18 250);
  --radius: 0.75rem;

screen Pricing:
  container:
    # Escolha seu plano
    grid-3:
      card:
        ## Starter
        @[Começar grátis](Signup)
      card-feature:
        ## Business
        @[Assinar Business](upgrade)
      card:
        ## Enterprise
        @[Falar com vendas](contact)
```
</DslPreview>
Quando usar: alinhar botões, cards e raio às diretrizes da marca.
Por quê: assegura que elementos críticos sigam o branding aprovado.

<DslPreview title="Tema escuro com contraste garantido">
```dsl
styles:
  --background: oklch(0.13 0.02 250);
  --foreground: oklch(0.95 0 0);
  --card: oklch(0.18 0.03 250);
  --muted: oklch(0.32 0.04 250);

screen DarkDashboard:
  container-wide:
    # Métricas noturnas
    grid-3:
      card:
        ## Leads
        # 1.240
      card:
        ## Conversões
        # 134
      card:
        ## Satisfação
        # 92%
```
</DslPreview>
Quando usar: oferecer modo escuro ou alto contraste sem alterar cada componente.
Por quê: melhora acessibilidade e cansaço visual com poucas mudanças.

<DslPreview title="Tokens customizados combinados com componentes">
```dsl
styles:
  --brand-header: #0b132b;
  --brand-subtitle: #5bc0be;

component Hero:
  card:
    ## %title
    > %subtitle
    @[Começar agora](%destination)

screen Landing:
  container:
    $Hero:
      - Proto-Typed | Ajuste o tema com CSS variables | Signup
```
</DslPreview>
Quando usar: criar tokens específicos e referenciá-los dentro de componentes compostos.
Por quê: facilita reaproveitamento dos mesmos valores em diferentes blocos.

## Erros comuns (Do/Don’t)
**Não faça — alterar tokens diretamente no componente**
<DslPreview title="❌ Errado — Estilo local sem tema">
```dsl
card:
  ## Configurações
  --primary: #2563eb;
  > Ajustes rápidos.
```
</DslPreview>

Faça — centralize no tema
<DslPreview title="✅ Correto — Token global aplicado">
```dsl
styles:
  --primary: #2563eb;

card:
  ## Configurações
  > Ajustes rápidos.
```
</DslPreview>
Por quê: tokens globais evitam divergências entre telas.

**Não faça — escolher cores sem contraste adequado**
<DslPreview title="❌ Errado — Baixo contraste">
```dsl
styles:
  --background: oklch(0.9 0.01 240);
  --foreground: oklch(0.88 0.01 240);
```
</DslPreview>

Faça — valide contraste mínimo
<DslPreview title="✅ Correto — Contraste forte">
```dsl
styles:
  --background: oklch(0.14 0.01 240);
  --foreground: oklch(0.95 0 0);
```
</DslPreview>
Por quê: contraste insuficiente compromete leitura e acessibilidade.

## Referências cruzadas

### Relacionados
- [`Styles Block`](./styles-block): local onde os tokens são declarados.
- [`Button`](./button): responde diretamente às variáveis de cor.
- [`Card`](./card): utiliza tokens de fundo e raio do tema.
- [`Dashboard`](./dashboard): demonstra a aplicação de temas em telas analíticas.

### Composições recomendadas
- **Themes + Styles Block** → separa tokens padrão e variações específicas.
- **Themes + Component Composition** → garante consistência em componentes complexos.
- **Themes + Modal** → controla contraste em overlays.

<DslPreview title="Tema aplicado junto com componente composto">
```dsl
styles:
  --primary: #1f2937;
  --primary-foreground: #f8fafc;

component HeroBanner:
  card:
    ## %title
    > %subtitle
    @[Começar agora](%action)

screen Home:
  container:
    $HeroBanner:
      - Lançamento do novo tema
      - Ajuste tokens para alinhar com a marca.
      - ThemeGuide
```
</DslPreview>
Por quê: o tema global define aparência enquanto o componente compõe conteúdo reutilizável.
