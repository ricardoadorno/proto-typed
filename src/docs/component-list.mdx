# Component List
## Por que importa
`list $Componente:` transforma dados tabulares em componentes reutilizáveis. Esse padrão garante consistência de layout ao renderizar dezenas de itens com poucas linhas de código.

## Forma canônica (sintaxe mínima)
<DocsCodePreview title="Lista baseada em componente">
```dsl
component ItemRow:
  row-between:
    > %title
    @[Abrir](%destination)

list $ItemRow:
  - Métricas | Metrics
  - Usuários | Users
```
</DocsCodePreview>
Por quê: mostra a relação direta entre linha de dados e placeholders usados no componente.

## Padrões de uso
<DocsCodePreview title="Catálogo de projetos">
```dsl
component ProjectCard:
  card:
    ## %name
    >>> %owner
    > Status: %status
    row-end:
      @outline[Detalhes](%destination)

list $ProjectCard:
  - Nova plataforma | Ana Souza | Em QA | ProjectDetail
  - Redesign mobile | Bruno Lima | Em desenvolvimento | MobileDetail
  - Pesquisa usuários | Carla Dias | Em discovery | ResearchDetail
```
</DocsCodePreview>
Quando usar: listar itens com múltiplos campos que compartilham o mesmo layout.
Por quê: garante consistência visual mesmo com grandes volumes de dados.

<DocsCodePreview title="Itens de menu em drawer">
```dsl
component NavItem:
  card-compact:
    ## %label
    >>> %description

drawer QuickMenu:
  container:
    # Seções
    list $NavItem:
      - Dashboard | Acompanhe métricas principais
      - Tarefas | Revise backlog e prioridades
      - Configurações | Ajuste preferências
```
</DocsCodePreview>
Quando usar: criar catálogos curtos com contexto adicional em painéis laterais.
Por quê: o drawer recebe dados estruturados sem duplicar markup em cada item.

<DocsCodePreview title="Documentos com ação de download">
```dsl
component DocumentRow:
  row-between:
    stack-tight:
      > %title
      >>> Atualizado em %updated
    @outline[i-Download Baixar](%url)

list $DocumentRow:
  - Guia rápido | 15/09 | https://example.com/quickstart.pdf
  - Release notes | 10/09 | https://example.com/release.pdf
```
</DocsCodePreview>
Quando usar: renderizar coleções com ação associada a cada item.
Por quê: centraliza botões dentro do componente, mantendo o código da lista limpo.

## Erros comuns (Do/Don’t)
<DoDont
  dont={{
    title: "Valores embaralhados",
    code: `
list $ProjectCard:
  - Nova plataforma | Ana Souza | Em QA | ProjectDetail
  - Em desenvolvimento | Bruno Lima | Redesign mobile | MobileDetail
    `,
  }}
  do={{
    title: "Ordem consistente",
    code: `
list $ProjectCard:
  - Nova plataforma | Ana Souza | Em QA | ProjectDetail
  - Redesign mobile | Bruno Lima | Em desenvolvimento | MobileDetail
    `,
  }}
>
  Por quê: a posição de cada valor deve corresponder ao `%prop` esperado pelo componente.
</DoDont>
<DoDont
  dont={{
    title: "Ação embutida nos dados",
    code: `
list $ItemRow:
  - Métricas @[Abrir](Metrics)
    `,
  }}
  do={{
    title: "Ação declarada no template",
    code: `
component ItemRow:
  row-between:
    > %title
    @[Abrir](%destination)

list $ItemRow:
  - Métricas | Metrics
    `,
  }}
>
  Por quê: manter a lógica no componente evita repetições e facilita mudanças futuras.
</DoDont>
