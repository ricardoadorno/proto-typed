# Component List

## O que é
Padrão que usa `list $Component:` para renderizar várias instâncias de um componente baseado em dados tabulares (pipes `|`).

## Capacidades
- Declarar `component` com `%props` e alimentá-lo via `list $Nome:`.
- Manter cada linha de dados limpa, fácil de atualizar ou gerar automaticamente.
- Misturar colunas com ações (`@button`, `#[Link]`) dentro do componente.

## Sintaxe
- `component ItemRow:`
- `row-between:`
- `> %title`
- `@[Detalhes](%destination)`
- `list $ItemRow:`
- `- Métricas | Metrics`
- `- Usuários | Users`


## Variantes & Slots
| Variante/Slot | Papel | Notas |
| --- | --- | --- |
| `component Nome` | Template | Contém `%props` usados na lista. |
| `list $Nome` | Coleção | Cada linha `-` envia valores posicionais. |
| `- valor1 | valor2` | Linha de dados | Ordem deve casar com `%props` declarados. |

## Casos de uso
- Listar cards, linhas de tabela ou itens de menu com dados dinâmicos.
- Criar catálogos com dados vindo de APIs ou planilhas (fácil de sincronizar).
- Reaproveitar o mesmo componente em dashboards e drawers.

## Exemplos
<DslPreview title="Exemplo 1">
```dsl
component ProjectCard:
  card:
    ## %name
    >>> %owner
    > Status: %status
    row-end:
      @outline[Detalhes](%destination)

screen Projects:
  container:
    # Projetos ativos
    list $ProjectCard:
      - Nova plataforma | Ana Souza | Em QA | ProjectDetail
      - Redesign mobile | Bruno Lima | Em desenvolvimento | MobileDetail
      - Pesquisa usuários | Carla Dias | Em discovery | ResearchDetail
```
</DslPreview>

<DslPreview title="Exemplo 2">
```dsl
component DocumentRow:
  row-between:
    stack:
      > %title
      >>> Atualizado em %updated
    @outline[i-Download Baixar](%url)

screen Files:
  container-narrow:
    # Arquivos disponíveis
    list $DocumentRow:
      - Guia rápido | 15/09 | https://example.com/quickstart.pdf
      - Release notes | 10/09 | https://example.com/release.pdf
      - Plano estratégico | 05/09 | https://example.com/plan.pdf
```
</DslPreview>
