# Drawer
Drawers exibem painéis laterais sobre a tela atual para oferecer filtros, detalhes ou ações auxiliares sem quebrar o contexto.

## Quando usar
- Expor filtros avançados enquanto a tela principal permanece visível.
- Mostrar detalhes de um item selecionado sem navegar para outra rota.
- Oferecer menus contextuais quando o espaço horizontal está disponível.
- Complementar fluxos que precisam de feedback rápido após uma ação principal.

## Exemplo básico
<DslPreview title="Drawer de filtros em dashboard">
```dsl
drawer FilterDrawer:
  container:
    # Filtros avançados
    ___select: Status{Todos | Em andamento | Concluído}
    ___select: Responsável{Ana | Bruno | Carla}
    row-end:
      @ghost[Cancelar](close)
      @[Aplicar filtros](applyFilters)

screen Dashboard:
  container-wide:
    # Visão geral
    @[Abrir filtros](FilterDrawer)
    grid-3:
      card:
        ## Leads
        # 1.240
      card:
        ## Conversões
        # 134
      card:
        ## Receita
        # R$ 87K
```
</DslPreview>
Por quê: mantém indicadores visíveis enquanto a pessoa ajusta critérios no painel lateral.

## Variantes

    # Configurações rápidas
    ___switch: Notificações{Ativado}

      @ghost[Fechar](close)
      @[Salvar ajustes](saveSettings)

</DslPreview>
Quando usar: organizar controles simples com espaçamento consistente.
Por quê: distribui ajustes rápidos no painel sem poluir a tela principal.

## Composição (usando com outros)
- **Drawer + Button** — aciona o painel a partir de CTAs contextuais.
- **Drawer + List** — apresenta coleções resumidas com ações fora da lista.
- **Drawer + Screen principal** — mantém fluxo contínuo sem redirecionar.

<DslPreview title="Tela com drawer de detalhes">
```dsl
screen Accounts:
  container-wide:
    # Contas ativas
    list:
      - Grupo Norte | @[Ver detalhes](AccountDrawer)
      - Grupo Sul | @[Ver detalhes](AccountDrawer)

drawer AccountDrawer:
  card:
    ## Grupo Norte
    > Dados atualizados do cliente.
    row-end:
      @ghost[Fechar](close)
      @[Editar conta](AccountEdit)
```
</DslPreview>
Por quê: permite investigar dados sem sair da lista principal.

## Slots & Props
| Nome | Tipo | Padrão | Descrição | Obrigatório |
| --- | --- | --- | --- | --- |
| `nome` | identificador | — | Define o slug do drawer (PascalCase recomendado). | Sim |
| `conteúdo interno` | blocos DSL | — | Recebe stacks, containers, cards, listas e formulários. | Sim |
| `ações de fechamento` | botões | — | Normalmente `@ghost[Fechar](close)` ou `@ghost[Cancelar](-1)`. | Sim |
| `acionadores` | botões/links | — | Elementos que chamam o drawer (`@[Abrir](DrawerName)`). | Sim |

## Erros comuns (Do/Don’t)
**Não faça — menus com `list` e links diretos**
<DslPreview title="❌ Errado — Lista com links navegáveis">
```dsl
drawer MainMenu:
  list:
    - #[Dashboard](Dashboard)
    - #[Relatórios](Reports)
```
</DslPreview>

Faça — use `navigator` ou botões fora de `list`
<DslPreview title="✅ Correto — Navegação estruturada">
```dsl
drawer MainMenu:
  navigator Menu:
    - Dashboard | Dashboard
    - Relatórios | Reports
```
</DslPreview>
Por quê: menus interativos devem usar `navigator` para manter semântica correta.

**Não faça — misturar `close` e `-1` no mesmo drawer**
<DslPreview title="❌ Errado — Fechamentos inconsistentes">
```dsl
drawer ConfirmDrawer:
  row-end:
    @ghost[Cancelar](-1)
    @[Salvar](close)
```
</DslPreview>

Faça — escolha um padrão de fechamento
<DslPreview title="✅ Correto — Padrão consistente">
```dsl
drawer ConfirmDrawer:
  row-end:
    @ghost[Cancelar](close)
    @[Salvar](saveDrawer)
```
</DslPreview>
Por quê: manter um único padrão evita confusão sobre o comportamento do fechamento.

## Referências cruzadas

### Relacionados
- [`Modal`](./modal): variação sobreposta para confirmações rápidas.
- [`Navigator`](./navigator): constrói menus e abas para navegação lateral.
- [`Drawer Detail`](./drawer-detail): guia com receita completa para detalhes no drawer.
- [`Button`](./button): principal acionador para abrir drawers.

### Composições recomendadas
- **Drawer + Button** → painel acionado sob demanda mantendo contexto.
- **Drawer + List** → exibe resumos enquanto a tela principal fica visível.
- **Drawer + Modal** → combine quando precisar confirmar ações iniciadas no drawer.

<DslPreview title="Fluxo drawer + modal">
```dsl
drawer BillingDrawer:
  card:
    ## Atualizar plano
    > Escolha o plano desejado e confirme a mudança.
    row-end:
      @ghost[Cancelar](close)
      @[Avançar](ConfirmBilling)

modal ConfirmBilling:
  card:
    # Confirmar atualização
    > A alteração entra em vigor imediatamente.
    row-end:
      @ghost[Voltar](close)
      @success[Confirmar](applyBilling)
```
</DslPreview>
Por quê: inicia ajustes no drawer e confirma em modal separado para reforçar a decisão.
