# Stack
Stacks empilham elementos verticalmente, controlando o espaçamento entre blocos de conteúdo.

## Quando usar
- Organizar títulos, textos e ações em sequência lógica.
- Montar formulários com múltiplos campos e botões alinhados.
- Alternar densidade de leitura com variantes `tight`, `loose` e `flush`.
- Agrupar componentes reutilizáveis mantendo hierarquia visual coesa.

## Exemplo básico
<DocsCodePreview title="Stack em card de orientações">
```dsl
screen Default:
  container:
    card:
      ## Próximos passos
      stack-loose:
        > Revise o backlog com o time.
        > Confirme o plano da sprint.
        @[Abrir planejamento](Planning)
```
</DocsCodePreview>
Por quê: combina textos e ação em blocos verticais com espaçamento confortável.

<Attention title="Evite hífens automáticos em layouts">
Mesmo que um LLM sugira `-` antes de cada linha, `stack`, `row`, `card` e demais layouts esperam apenas filhos indentados. Use hífen somente em `list`, `navigator` e `fab` para impedir que a sintaxe gere nós inválidos.
</Attention>

## Variantes
<DocsCodePreview title="Stack padrão">
```dsl
screen Default:
  container:
    stack:
      > Valide os requisitos
      > Ajuste o cronograma
      @[Abrir checklist](Checklist)
```
</DocsCodePreview>
Quando usar: organizar conteúdos variados com espaçamento mediano.
Por quê: equilibra textos, inputs e ações sem gerar blocos soltos.

<DocsCodePreview title="Stack-tight para itens curtos">
```dsl
screen Default:
  container:
    stack-tight:
      [X] Definir objetivo
      [ ] Validar protótipo
      [ ] Preparar release
```
</DocsCodePreview>
Quando usar: agrupar itens curtos ou checklists sem perder legibilidade.
Por quê: o gap reduzido aproxima conteúdos relacionados e acelera leitura.

<DocsCodePreview title="Stack-flush para cards compactos">
```dsl
screen Default:
  container:
    stack-flush:
      card-compact:
        > Última atualização em 18/09.
      card-compact:
        > Reunião agendada para amanhã.
```
</DocsCodePreview>
Quando usar: empilhar elementos que já possuem padding próprio.
Por quê: evita espaçamento duplo quando os componentes internos já trazem margem.

## Composição (usando com outros)
- **Stack + Card** — coloca conteúdos sequenciais dentro de um bloco estruturado.
- **Stack + Row** — mistura distribuição vertical e horizontal na mesma área.
- **Stack + Component Definition** — reutiliza blocos padronizados em várias telas.

<DocsCodePreview title="Formulário com stack e row">
```dsl
screen Default:
  container:
    card:
      ## Atualizar perfil
      stack-loose:
        ___: Nome completo{Digite seu nome}
        ___email: E-mail{nome@empresa.com}
        ___textarea: Bio{Conte sobre seu trabalho}
      row-end:
        @ghost[Cancelar](close)
        @[Salvar mudanças](saveProfile)
```
</DocsCodePreview>
Por quê: separa campos verticalmente e alinha ações no rodapé.

## Slots & Props
| Nome | Tipo | Padrão | Descrição | Obrigatório |
| --- | --- | --- | --- | --- |
| `stack` | layout vertical | `stack` | Empilha elementos com espaçamento padrão. | Não |
| `stack-tight`, `stack-loose`, `stack-flush` | variações | — | Ajustam densidade entre itens (compacto, amplo, sem gap). | Não |
| `conteúdo interno` | blocos DSL | — | Itens empilhados, como textos, inputs e componentes. | Sim |

## Erros comuns (Do/Don’t)
<DoDont
  dont={{
    title: "Stack com botões lado a lado",
    code: `
stack:
  @[Cancelar](close)
  @[Salvar](saveForm)
    `,
  }}
  do={{
    title: "Ações horizontais",
    code: `
stack:
  > Ajuste as preferências antes de salvar.
  row-end:
    @ghost[Cancelar](close)
    @[Salvar](saveForm)
    `,
  }}
>
  Por quê: stacks tratam a ordem vertical; rows cuidam de alinhamento horizontal.
</DoDont>
<DoDont
  dont={{
    title: "Textos longos sem respiro",
    code: `
stack-tight:
  > Atualize relatórios de todas as squads com detalhes completos.
  > Documente aprendizados em um relatório amplo para todo o time.
    `,
  }}
  do={{
    title: "Espaço adequado",
    code: `
stack-loose:
  > Atualize relatórios de todas as squads com detalhes completos.
  > Documente aprendizados em um relatório amplo para todo o time.
    `,
  }}
>
  Por quê: conteúdo amplo precisa de espaçamento maior para manter a leitura confortável.
</DoDont>

## Referências cruzadas

### Relacionados
- [`Row`](./row): complementa stacks quando é preciso alinhar ações na horizontal.
- [`Card`](./card): envolve stacks com título e contexto.
- [`Container`](./container): estrutura principal antes de incluir stacks.
- [`Component Definition`](./component-definition): cria blocos reutilizáveis empilhados.

### Composições recomendadas
- **Stack + Row-end** → formulários com ações alinhadas.
- **Stack + Card + List** → seções de conteúdo com resumos textuais.
- **Stack + Component Definition** → layouts reutilizáveis com dados variáveis.

<DocsCodePreview title="Stack com componente reutilizável">
```dsl
component Insight:
  card:
    ## %title
    > %description

screen Default:
  container:
    stack-loose:
      $Insight:
        - Roadmap atualizado | Planeje entregas para o próximo trimestre.
      $Insight:
        - Métricas semanais | Revise KPIs antes da reunião de status.
```
</DocsCodePreview>
Por quê: evita duplicação ao empilhar componentes com dados diferentes.
