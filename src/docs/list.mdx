# List

## O que é
Estrutura vertical que organiza itens em sequência, com suporte a texto simples ou instâncias de componentes reutilizáveis.

## Capacidades
- Declarar listas básicas com `list:` + itens `-`.
- Renderizar componentes com props usando `list $Component:`.
- Passar dados por linha via pipes (`- valor1 | valor2`) e interpolar com `%prop`.
- Mesclar ações inline (botões `@variant` ou links `#[...]`).
- Combinar com `component` para criar catálogos, menus e checklists dinâmicos.

## Sintaxe
- `list:` seguido apenas de linhas `- texto simples`
- `component Nome:` para preparar `%props`
- `list $Nome:` com linhas `- valor1 | valor2` que preenchem `%props`

## Sintaxe inválida
- `list:` com `- Item | Outro valor` (use `list $Componente` ou `card`)
- `list:` com `- Texto @[Ação](Destino)` (envolva em `card`/`stack`)

## Variantes & Slots
| Variante/Slot | Papel | Notas |
| --- | --- | --- |
| `list` | Lista simples | Cada item inicia com `-`. |
| `list $Name` | Lista baseada em componente | Requer um `component Name:` declarado previamente. |
| `- valor | valor` | Linha com props | A ordem deve corresponder aos `%props`. |
| Conteúdo inline | Slot | Permite botões, links e formatos mistos nos itens. |

## Casos de uso
- `list:` Mostrar tarefas com status ou datas.
- `list $Component:` Montar cards repetidos a partir de um template.
- `list:` dentro de `drawer` ou `navigator` para menus laterais.
- **Cuidado:** itens da lista aceitam apenas texto; para conteúdo avançado (cards, botões, múltiplas colunas) use `card`, `stack` ou `list $Componente`.

## Exemplos
<DslPreview title="Exemplo 1">
```dsl
component TaskRow:
  row-between:
    stack:
      > %title
      >>> %due
    row:
      @outline-sm[i-Edit Editar](edit:%id)
      @destructive-sm[i-Trash Remover](ConfirmDelete)

screen Tasks:
  container:
    # Tarefas da sprint
    list $TaskRow:
      - Atualizar documentação | Vence em 20/09 | 101
      - Revisar backlog | Vence em 22/09 | 102
      - Preparar demo | Vence em 25/09 | 103

  container:
    stack:
      card-compact:
        ## Checklist
        @[Abrir checklist](Checklist)
      card-compact:
        ## Exportar relatório
        @[Exportar CSV](https://example.com/export.csv)
```
</DslPreview>

<DslPreview title="Exemplo 2">
```dsl
drawer QuickMenu:
  container:
    # Menu rápido
    stack:
      card-compact:
        ## Dashboard
        #[Abrir dashboard](Dashboard)
      card-compact:
        ## Tarefas
        #[Ir para tarefas](Tasks)
      card-compact:
        ## Configurações
        #[Abrir preferências](Settings)

screen Dashboard:
  header:
    # Proto-Typed
    @ghost[i-Menu Menu](QuickMenu)

  container:
    # Visão geral
    stack:
      card:
        ## Releases recentes
        @[Ver changelog](https://example.com/changelog)
      card:
        ## Próxima reunião
        @[Abrir agenda](Calendar)

modal ConfirmDelete:
  card:
    # Remover tarefa
    > Esta ação não pode ser desfeita.
    row-end:
      @ghost[Cancelar](close)
      @destructive[Excluir agora](deleteTask)
```
</DslPreview>
