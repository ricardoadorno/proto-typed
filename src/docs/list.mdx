# List
Listas organizam itens textuais em sequência vertical, servindo como ponto de partida para resumos e catálogos.

## Quando usar
- Exibir coleções de itens curtos com leitura rápida.
- Estruturar resumos, agendas ou checklists sem formatação complexa.
- Alimentar componentes repetidos (`list $Componente`) com dados tabulares.
- Agrupar conteúdos em drawers, modais ou cards mantendo ritmo vertical.

<Attention title="Indentação primeiro, hífen apenas em listas">
LLMs costumam inserir `-` para listar filhos de qualquer bloco, mas a DSL só aceita esse marcador em `list`, `navigator` e `fab`. Para layouts e componentes use apenas indentação; isso evita nós fantasmas e mantém a leitura clara para quem dá manutenção.
</Attention>

## Exemplo básico
<DslPreview title="Lista com ações separadas">
```dsl
card:
  ## Tarefas da semana
  list:
    - Revisar backlog
    - Preparar demo
    - Atualizar documentação
  row-end:
    @[Criar nova tarefa](TaskForm)
    @ghost[Ver calendário](Calendar)
```
</DslPreview>
Por quê: mantém itens exclusivamente textuais e posiciona ações em bloco dedicado.

## Variantes
<DslPreview title="Lista simples">
```dsl
list:
  - Configurar métricas
  - Validar integrações
  - Revisar deploy
```
</DslPreview>
Quando usar: enumerar itens curtos sem propriedades adicionais.
Por quê: apresenta conteúdos simples sem exigir componentes complexos.

<DslPreview title="Lista baseada em componente">
```dsl
component TaskRow:
  row-between:
    stack-tight:
      > %title
      >>> %due
    @outline-sm[Detalhar](%action)

list $TaskRow:
  - Atualizar roadmap | Até 22/09 | show:roadmap
  - Revisar backlog | Até 24/09 | show:backlog
```
</DslPreview>
Quando usar: renderizar itens repetidos com múltiplos campos seguindo a mesma estrutura.
Por quê: evita duplicação de markup enquanto mantém layout consistente.

<DslPreview title="Lista compacta em drawer">
```dsl
drawer QuickTips:
  container:
    # Dicas rápidas
    list:
      - Use filtros por período para refinar métricas.
      - Combine stack e card para formular layouts.
      - Consulte o guia de componentes antes do handoff.
```
</DslPreview>
Quando usar: oferecer notas rápidas sem ocupar muito espaço.
Por quê: o drawer traz lembretes sucintos preservando a área principal.

## Composição (usando com outros)
- **List + Card** — agrupa contexto e mantém itens alinhados.
- **List + Component Definition** — transforma dados tabulares em layouts ricos.
- **List + Drawer** — expõe listas de apoio sem sair da tela principal.

<DslPreview title="Lista com componente reutilizável">
```dsl
component MetricItem:
  row-between:
    > %label
    >> %value

card:
  ## Indicadores
  list $MetricItem:
    - Leads captados | 1.240
    - Conversões | 134
    - Receita | R$ 87K
```
</DslPreview>
Por quê: mantém consistência na apresentação de métricas recorrentes.

## Slots & Props
| Nome | Tipo | Padrão | Descrição | Obrigatório |
| --- | --- | --- | --- | --- |
| `list` | bloco | — | Renderiza itens textuais com marcador `-`. | Sim |
| `list $Componente` | bloco | — | Usa componente previamente declarado para cada linha. | Não |
| `itens` | texto ou dados separados por `|` | — | Dados passados para cada item ou componente. | Sim |
| `%props` | interpolação | — | Variáveis dentro do componente associado à lista. | Não |

## Erros comuns (Do/Don’t)
**Não faça — incluir links ou botões dentro da lista**
<DslPreview title="❌ Errado — Ações em `list:`">
```dsl
list:
  - Revisar backlog @[Editar](TaskForm)
```
</DslPreview>

Faça — mova as ações para blocos de ação
<DslPreview title="✅ Correto — Ações separadas">
```dsl
list:
  - Revisar backlog
  - Preparar demo
row-end:
  @[Editar tarefa](TaskForm)
```
</DslPreview>
Por quê: `list` aceita somente texto; ações ficam em rows, cards ou navigator.

**Não faça — usar `|` sem componente associado**
<DslPreview title="❌ Errado — Pipes sem `list $Componente`">
```dsl
list:
  - Plano Pro | R$ 89
```
</DslPreview>

Faça — declare componente para receber props
<DslPreview title="✅ Correto — Lista com componente">
```dsl
component PlanRow:
  row-between:
    > %title
    >> %price

list $PlanRow:
  - Plano Pro | R$ 89
```
</DslPreview>
Por quê: `|` indica múltiplos valores, e apenas componentes conseguem interpolar `%props`.

## Referências cruzadas

### Relacionados
- [`Component Definition`](./component-definition): define templates usados em `list $`.
- [`Component List`](./component-list): amplia listas com dados dinâmicos.

- **List + Card + Row-end** → resumo textual com ações no rodapé.
- **List + Component Definition** → coleções ricas baseadas em dados.
- **List + Drawer** → notas rápidas sem navegar de tela.
