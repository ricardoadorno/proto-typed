# Component Props

## O que é
Mecanismo para passar valores dinâmicos a componentes via linhas com pipes (`- valor1 | valor2`) e placeholders `%prop` no template.

## Capacidades
- Substituir `%prop` por valores posicionais definidos na instância.
- Trabalhar com múltiplas colunas em `list $Component:` ou instâncias individuais `$Component:`.
- Combinar props com ações, ícones e textos dentro do componente.

## Sintaxe
- `component UserCard:`
- `card:`
- `## %name`
- `>> %email`
- `@[Contato](mailto:%email)`
- `screen Team:`
- `list $UserCard:`
- `- Ana Souza | ana@empresa.com`
- `- Bruno Lima | bruno@empresa.com`


## Variantes & Slots
| Variante/Slot | Papel | Notas |
| --- | --- | --- |
| `%prop` | Placeholder | Define posição da informação na instância. |
| `$Componente:` | Instância com props | Cada linha `-` mapeia para os `%props`. |
| `list $Componente:` | Coleção | Usa várias linhas de dados. |

## Casos de uso
- Criar cards reutilizáveis com dados dinâmicos (usuários, produtos, métricas).
- Montar listas repetitivas em `list $Component:` sem duplicar markup.
- Gerar dashboards com múltiplas props (título, valor, tendência).

## Exemplos
<DslPreview title="Exemplo 1">
```dsl
component UserCard:
  card:
    ## %name
    >> %role
    >>> %email
    @[Enviar mensagem](mailto:%email)

screen Team:
  container:
    # Nossa equipe
    list $UserCard:
      - Ana Souza | Product Manager | ana@empresa.com
      - Bruno Lima | Engineering Manager | bruno@empresa.com
      - Carla Dias | UX Designer | carla@empresa.com
```
</DslPreview>

<DslPreview title="Exemplo 2">
```dsl
component MetricCard:
  card:
    >>> %label
    # %value
    >> %trend

screen Dashboard:
  container-wide:
    grid-3:
      $MetricCard:
        - Leads | 1.240 | +8%
      $MetricCard:
        - Conversões | 134 | +5%
      $MetricCard:
        - Satisfação | 92% | +3%
```
</DslPreview>
