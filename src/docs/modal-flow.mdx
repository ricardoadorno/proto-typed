# Modal Flow

## O que é
Padrão de fluxo que coordena múltiplos modais para confirmar ações ou exibir etapas sequenciais sem trocar de tela.

## Capacidades
- Encadear modais (`modal Confirm`, `modal Sucesso`) acionados por botões ou FAB.\n+- Reutilizar o mesmo modal a partir de diferentes pontos (lista, card, FAB).\n+- Direcionar usuários rapidamente por confirmações e feedbacks sem sair da `screen` atual.

## Sintaxe
- `modal ConfirmInvite:`
- `card:`
- `# Enviar convite?`
- `row-end:`
- `@ghost[Cancelar](close)`
- `@[Enviar convite](SuccessInvite)`
- `modal SuccessInvite:`
- `card:`
- `# Convite enviado!`
- `@[Fechar](close)`
- `screen Team:`
- `container:`
- `@[Convidar membro](ConfirmInvite)`


## Variantes & Slots
| Variante/Slot | Papel | Notas |
| --- | --- | --- |
| `modal Confirm...` | Etapa de confirmação | Deve oferecer ação de saída segura (`close`). |
| `modal Success...` | Feedback rápido | Pode exibir CTA secundário ou apenas fechar. |
| Botões de acionamento | Entrada do fluxo | Aparecem em `list`, `card`, `fab`, etc. |

## Casos de uso
- Confirmar convites ou compartilhamentos antes de enviar.\n+- Alertar usuários sobre ações destrutivas antes da execução.\n+- Mostrar feedback positivo/erro logo após a confirmação.

## Exemplos
<DslPreview title="Exemplo 1">
```dsl
modal ConfirmInvite:
  card:
    # Enviar convite?
    > Confirme o envio para o colaborador selecionado.
    row-end:
      @ghost[Cancelar](close)
      @[Enviar convite](SuccessInvite)

modal SuccessInvite:
  card:
    # Convite enviado!
    > O usuário receberá um email com instruções de acesso.
    @[Fechar](close)

screen Team:
  container:
    # Equipe
    card:
      ## Convites recentes
      list:
        - Thiago Martins | Enviado em 16/09 | @[Reenviar](ConfirmInvite)
        - Laura Mendes | Pendente | @[Enviar agora](ConfirmInvite)

  fab:
    - i-Plus | ConfirmInvite
```
</DslPreview>

<DslPreview title="Exemplo 2">
```dsl
modal ConfirmDelete:
  card:
    # Confirmar exclusão
    > Esta ação não pode ser desfeita.
    row-end:
      @ghost[Cancelar](close)
      @destructive[Excluir](SuccessDelete)

modal SuccessDelete:
  card:
    # Registro removido
    > Os dados foram apagados com sucesso.
    @[Fechar](close)

screen DangerZone:
  container:
    # Zona de risco
    list:
      - Deletar workspace | @[Remover](ConfirmDelete)
      - Revogar acesso | @[Revogar](ConfirmDelete)
```
</DslPreview>
