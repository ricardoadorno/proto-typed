# Link

## O que é
Links internos e externos com semântica clara, usados para navegar entre telas ou apontar recursos relevantes.

## Sintaxe
```proto
#[Texto do link](Destino)
```

Exemplo
<DslPreview>
```dsl
screen Support:
  container:
    # Central de Ajuda
    > Resolva dúvidas consultando os materiais oficiais.

    stack:
      #[Ver FAQs](https://proto-typed.dev/faq)
      #[Abrir Dashboard](Dashboard)
      #[Enviar email](mailto:support@proto-typed.dev)

screen Dashboard:
  container:
    # Visão Geral
    > Última sincronização em 18/09 às 10:15.
    #[Voltar para suporte](Support)
```
</DslPreview>

## Variações e Props
| Prop | Tipo | Descrição | Exemplo |
| --- | --- | --- | --- |
| `href` | string | Destino interno (nome de tela), externo (URL) ou esquema (`mailto`, `tel`) | `#[Contato](mailto:suporte@...)` |
| `target` | atributo | Adicione `| target="_blank"` para abrir em nova aba | `#[Ver site](https://... ) | target="_blank"` |
| `variant` | string | Combine com estilos usando `@link` quando precisar comportamento de botão | `@link[Ajuda](Support)` |
| `inline` | contexto | Links podem ser usados dentro de texto com markdown | `> Consulte #[Guia de estilos](Styles)` |
| `overlay` | literal | Chamadas para modal/drawer usam o nome do overlay como destino | `#[Abrir filtros](FilterDrawer)` |

## Boas práticas
- Prefira links para navegação contextual e botões para ações primárias.
- Indique claramente quando o link leva a recurso externo ou baixa arquivos.
- Agrupe links relacionados em `list` para manter estrutura limpa.

## Erros comuns
⚠ Esquecer os colchetes ou parênteses quebra o link (`#[Texto](Destino)`).
⚠ Misturar links e botões sem padrão dificulta a leitura da interface.
