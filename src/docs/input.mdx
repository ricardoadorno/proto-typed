# Input

## O que é
Tokens de formulário usados para capturar dados do usuário em campos de texto, e-mail, senha, números, datas ou áreas de texto.

## Capacidades
- Declarar campos com `___type?: Label{placeholder}`.
- Definir placeholders, opções (`[Opção A | Opção B]`) e atributos adicionais após `|`.
- Registrar checkboxes `[X] Label` e radios `( ) Label` diretamente na DSL.
- Compor formulários em `card`, `stack` ou `grid`, alinhando botões com `row-end`.

## Sintaxe
- `___: Nome{Seu nome}`
- `___email: Email{email@dominio.com}`
- `___password: Senha{mínimo 8 caracteres} | required minlength="8"`
- `___number: Quantidade{0}`
- `___date: Data{Selecione}`
- `___textarea: Descrição{Detalhes adicionais}`
- `[X] Aceito os termos`
- `( ) Plano mensal`


## Variantes & Slots
| Variante/Slot | Papel | Notas |
| --- | --- | --- |
| `___` (text) | Campo padrão | Texto livre. |
| `___email`, `___password`, `___number`, `___date`, `___textarea` | Tipos específicos | Validam formato ou ampliam área de edição. |
| `[X] / [ ]` | Checkbox | Estado inicial definido pela presença do `X`. |
| `(X) / ( )` | Radio | Agrupar radios consecutivos para seleção única. |
| Atributos pós `|` | Customização | `required`, `minlength="8"`, `placeholder` extra. |

## Casos de uso
- `___email:` Capturar credenciais de login ou contato.
- `___textarea:` Receber descrições longas e feedbacks.
- `___number:` Registrar quantidades, estimativas ou valores.
- `[X] Aceito os termos` / `( ) Plano mensal`: Confirmar consentimentos e escolhas exclusivas.

## Exemplos
<DslPreview title="Exemplo 1">
```dsl
screen Signup:
  container:
    # Criar conta
    card:
      stack:
        ___: Nome completo{Seu nome}
        ___email: E-mail{usuario@empresa.com} | required
        ___password: Senha{Mínimo 8 caracteres} | required minlength="8"
        ___date: Início da assinatura{Selecione}
        ___textarea: Objetivos{Descreva como pretende usar o produto}
        [X] Aceito os termos de uso

      row-end:
        @ghost[Cancelar](-1)
        @[Criar conta](submitSignup)
```
</DslPreview>

<DslPreview title="Exemplo 2">
```dsl
component OptionRow:
  row-between:
    > %label
    @[Selecionar](%action)

screen Billing:
  container-narrow:
    # Configurar plano
    list $OptionRow:
      - Plano Starter | choose:starter
      - Plano Business | choose:business

    stack-tight:
      (X) Cobrança mensal
      ( ) Cobrança anual

    row-end:
      @ghost[Cancelar](-1)
      @primary[Salvar alterações](savePlan)
```
</DslPreview>
