# Styles Block

## O que é
Bloco `styles:` que permite definir e sobrescrever CSS custom properties utilizadas pelo renderer (cores, espaçamentos, fontes).

## Capacidades
- Declarar variáveis como `--primary`, `--accent`, `--radius` para personalizar tema.\n+- Criar tokens próprios (`--brand-color`, `--spacing-card`) e reutilizar no layout.\n+- Aplicar estilos globais que afetam botões, cards e tipografia.

## Sintaxe
- `styles:`
- `--primary: oklch(0.68 0.15 210);`
- `--primary-foreground: oklch(0.98 0 0);`
- `--radius: 0.75rem;`


## Variantes & Slots
| Variante/Slot | Papel | Notas |
| --- | --- | --- |
| `styles:` | Início do bloco | Válido no topo do arquivo DSL. |
| `--token: valor;` | Propriedade CSS | Deve terminar com `;`. |

## Casos de uso
- Ajustar cores primárias e secundárias de acordo com a identidade da marca.\n+- Configurar `--radius` para alterar bordas de componentes.\n+- Definir tokens customizados reutilizados em componentes (`--brand-color`). 

## Exemplos
<DslPreview title="Exemplo 1">
```dsl
styles:
  --primary: oklch(0.7 0.15 220);
  --primary-foreground: oklch(0.98 0 0);
  --accent: oklch(0.68 0.18 250);
  --radius: 0.75rem;

screen CustomTheme:
  container:
    # Tema personalizado
    > Botões e cards usam as variáveis definidas no bloco.
    row:
      @[Ação principal](Action)
      @secondary[Secundário](Secondary)
```
</DslPreview>

<DslPreview title="Exemplo 2">
```dsl
styles:
  --brand-color: #0f172a;
  --brand-subtitle: #cbd5f5;

component Hero:
  card:
    ## %title
    > %subtitle
    @[Começar agora](%destination)

screen Landing:
  container:
    $Hero:
      - Proto-Typed | Construa protótipos com velocidade | Signup
```
</DslPreview>
